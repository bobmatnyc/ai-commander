# Unified /connect Command for All Sessions

## Description
## Vision

Unify the /connect command to handle all tmux sessions, not just commander-managed ones.

## Current Behavior

- /connect only handles sessions with `commander-` prefix
- Regular tmux sessions are invisible to Commander
- Users cannot connect to pre-existing tmux sessions

## Target Behavior

- /connect shows unified list of ALL tmux sessions
- Session list distinguishes commander-managed vs regular sessions
- Users can connect to any tmux session regardless of origin
- Graceful handling of sessions without commander metadata

## Implementation Approach

1. Detect and list both session types
2. Update /connect to handle regular sessions
3. Add UI indicators for session type
