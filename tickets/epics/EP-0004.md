# Make Agent Orchestration Standard Behavior

## Description
## Vision

Make agent orchestration the default behavior across all Commander interfaces, not just the CLI `agent` subcommand.

## Current State

- Only `agent` CLI subcommand uses the AgentOrchestrator
- REPL, TUI, and Telegram bot all bypass orchestration
- This creates inconsistent behavior across interfaces

## Target State

All user interactions (REPL, TUI, Telegram, CLI) route through AgentOrchestrator by default, providing:
- Consistent agent delegation behavior
- Unified task management
- Multi-agent coordination across all interfaces

## Estimated Scope

- **Files affected**: ~8 files
- **Lines of code**: ~420 lines added
- **Timeline**: 4-5 weeks phased implementation

## Implementation Phases

1. Enable agents feature by default (quick win)
2. Auto-initialize orchestrator in TUI
3. Route TUI messages through orchestrator
4. Add orchestrator to REPL
5. Add orchestrator to Telegram bot
