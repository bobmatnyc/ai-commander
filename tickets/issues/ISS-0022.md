# Bug: Timer notifications passed through literally instead of being interpreted

## Description
Timer notifications like '[timer] 1 new session(s) waiting for input: @izzie-33 - ...' are being passed through to the agent literally instead of being parsed/interpreted.

**Expected behavior:**
- The agent should interpret these notifications
- Parse the session name, project path, branch status, model info, and context usage
- Take action based on the notification (e.g., switch attention to session needing input)
- Strip ANSI escape codes for clean display

**Actual behavior:**
- Raw text with escape codes passed through literally
- Agent does not understand or act on the notification

**Technical notes:**
The timer notification format includes:
- Session identifier (e.g., @izzie-33)
- User and host info (e.g., masa@Masas-Studio)
- Working directory path
- Git branch and status (e.g., main*?)
- ANSI escape codes for formatting
- Model info and context usage percentage
