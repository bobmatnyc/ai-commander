# Detect and list both commander and regular tmux sessions

## Description
## Summary

Modify session discovery to find all tmux sessions, not just commander-prefixed ones, and display a unified list with type indicators.

## Current Behavior

Session discovery only looks for sessions with `commander-` prefix:
```rust
// Only finds: commander-project-name
tmux list-sessions | grep "commander-"
```

## Target Behavior

Session discovery finds ALL tmux sessions and categorizes them:
```
Available Sessions:
  [C] commander-ai-commander (3 windows)
  [C] commander-web-app (1 window)
  [R] my-dev-session (2 windows)
  [R] logs (1 window)
```

Where:
- `[C]` = Commander-managed session
- `[R]` = Regular tmux session

## Implementation

1. Run `tmux list-sessions` without filtering
2. Parse session list to identify both types
3. Extract commander metadata where available
4. Create unified session list data structure
5. Display with type indicators

## Technical Details

- Location: Session discovery module
- Key change: Remove `commander-` prefix filter
- Add: Session type enum (Commander, Regular)

## Acceptance Criteria

- [ ] All tmux sessions are discovered
- [ ] Sessions are correctly categorized as Commander or Regular
- [ ] Unified list displays both types with clear indicators
- [ ] Session metadata extracted where available
