# Session type indicator in UI

## Description
## Summary

Add visual indicators in the UI to distinguish commander-managed sessions from regular tmux sessions.

## Current State

Session lists show all sessions identically without distinguishing their type.

## Target State

Session lists clearly indicate which sessions are commander-managed vs regular:

### Option A: Icon/Prefix Indicators
```
Sessions:
  [*] commander-ai-commander
  [*] commander-web-app
  [ ] my-dev-session
  [ ] logs
```

### Option B: Color Coding
- Commander sessions: Green
- Regular sessions: Gray/White

### Option C: Grouping
```
Commander Sessions:
  - ai-commander
  - web-app

Other Sessions:
  - my-dev-session
  - logs
```

## Implementation

1. Add session type to session list data
2. Update UI components to display indicators
3. Consider consistent indicators across TUI, REPL, Telegram
4. Add legend/help text explaining indicators

## UI Locations

- TUI session picker
- REPL `/connect` output
- Telegram session list messages

## Acceptance Criteria

- [ ] Session type is visually distinguishable
- [ ] Indicator style is consistent across interfaces
- [ ] Legend or help text explains indicators
- [ ] Indicators don't clutter the UI
