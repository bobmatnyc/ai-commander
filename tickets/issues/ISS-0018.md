# Add AgentOrchestrator to Telegram bot

## Description
## Summary

Integrate AgentOrchestrator into the Telegram bot interface to provide consistent orchestration across all communication channels.

## Current State

Telegram bot processes messages directly without orchestration capabilities.

## Target State

Telegram bot uses AgentOrchestrator for message processing, enabling:
- Multi-agent coordination via Telegram
- Task management and delegation
- Consistent behavior with other interfaces

## Implementation

1. Add orchestrator to Telegram bot state
2. Initialize orchestrator during bot startup
3. Route incoming messages through orchestrator
4. Format orchestrator responses for Telegram
5. Handle async orchestrator operations
6. Graceful error handling and fallback

## Technical Considerations

- Telegram's async message handling
- Message formatting differences
- Rate limiting and response chunking
- Session state management

## Estimated Scope

- ~200 lines of code
- Higher effort due to Telegram-specific formatting

## Acceptance Criteria

- [ ] Telegram bot initializes orchestrator on startup
- [ ] Incoming messages are processed by orchestrator
- [ ] Responses are properly formatted for Telegram
- [ ] Async operations handled correctly
- [ ] Error states provide user-friendly messages
