# Auto-initialize AgentOrchestrator in TUI

## Description
## Summary

Initialize the AgentOrchestrator automatically in `App::new()` instead of leaving it as `Option<None>`.

## Current State

TUI's `App` struct has `orchestrator: Option<AgentOrchestrator>` that defaults to `None` and is only initialized on demand.

## Target State

AgentOrchestrator is automatically initialized when the TUI starts, ready for use in message routing.

## Implementation Considerations

1. Handle async/sync boundary - AgentOrchestrator may require async initialization
2. Use tokio runtime to bridge async initialization in sync `App::new()`
3. Handle initialization failures gracefully
4. Consider lazy initialization if startup time is a concern

## Technical Details

- Location: `crates/commander-tui/src/app.rs`
- Key function: `App::new()`
- Dependency: tokio runtime for async initialization

## Effort

Medium effort - requires async/sync boundary handling

## Acceptance Criteria

- [ ] Orchestrator is initialized when TUI starts
- [ ] Initialization errors are handled gracefully
- [ ] TUI startup time is not significantly impacted
- [ ] Orchestrator is available for message routing
